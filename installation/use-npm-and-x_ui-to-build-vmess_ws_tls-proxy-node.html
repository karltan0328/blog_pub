设置x-ui

  1. 修改xray版本：在“系统状态”页，点击“xray状态”后的切换版本，直接切换为最新版本即可；

  2. 修改“面板url根路径”：点击“面板设置”页，首次进入该页会自动生成一个“面板url根路径”且会自动跳转到这个路径，你可以使用这个路径，但是后续的设置内容需要随之改变，这边推荐设置为/fuckgfw/，以XUI_PATH指代该路径；

  3. 添加入站：点击“入站列表”页，再点击“添加入站”：

    • 备注可以随便设置；
    • 协议改为vmess；
    • 端口可以改一个比较高的端口，这里以VMESS_PORT指代该端口；
    • 总流量可以根据你VPS的实际总流量-50进行设置；
    • 自动重置打开；
    • 点击添加用户后的第一个+添加一个用户；
    • 保存自动生成的User1 id，形式为：f61fxxxx-xxxx-xxxx-xxxx-5d75a33361e7，这里以VMESS_ID指代；
    • email可以自行设置，这里不重要；
    • 流量限制根据实际情况设置，在总流量处设置了问题就不大；
    • 网络选择ws；
    • 路径这边推荐为/fuckgfwvmess，这里以VMESS_PATH指代这个路径；
    • 其余都保持不动，点击保存。

设置Nginx Proxy Manager

如果没有任何设置，此时你登录Nginx Proxy Manager后，应该下面4项都显示为0：

  • 0 Proxy Hosts
  • 0 Redirection Hosts
  • 0 Streams
  • 0 404 Hosts

下面开始设置：

  1. 点击0 Proxy Hosts，此时Proxy Hosts列表应该为空；

  2. 点击右上角的Add Proxy Host；

  3. 去Cloudflare解析一个三级域名到你VPS的IP；

  4. 在Details页面中：

    1. 将这个三级域名填入Domain Names中；
    2. 在Forward Hostname/IP中填入172.17.0.1；
    3. 在Forward Port中填入81；

  5. 在Custom locations页面中：

    1. 点击Add location；
    2. 在Define location-location中填入XUI_PATH，在本教程中即/fuckgfw；
    3. 在Forward Hostname/IP中填入172.17.0.1；
    4. 在Forward Port中填入XUI_PORT；

  6. 在SSL页面中：

    1. 在SSL Certificate中选择Request a new SSL Certificate；
    2. 勾选Force SSL；
    3. 在Email Address for Let's Encrypt中填入邮箱；
    4. 勾选I Agree to the Let's Encrypt Terms of Service；

  7. 在Advanced页面中填入：

    注意修改对应的VMESS_PATH和VMESS_PORT；

    点击保存；

  8. 点击保存后，在Proxy Hosts列表中应该会出现一条记录，点击右侧的三个点进行编辑，再次打开SSL页中的Force SSL，设置结束，现在访问你在Cloudflare上解析到VPS的三级域名就可以访问Nginx Proxy Manager了。

设置Shadowrocket配置

  1. 点击应用右上角的+；
  2. 类型选择Vmess；
  3. 地址为你解析到VPS的三级域名；
  4. 端口443；
  5. UUID填VMESS_ID；
  6. 额外ID为0；
  7. 算法auto；
  8. 传输方式选websocket，Host为你解析到VPS的三级域名，路径为/VMESS_PATH；
  9. TLS开启；
  10. 多路复用关闭
  11. TCP快速打开关闭
  12. UDP转发开启；
  13. 代理通过无；
  14. 备注随意；
  15. 点击右上角的保存。

选中节点，进行测速，如果有测速结果，则节点搭建完成，可以访问https://ip.skk.moe/查看当前IP。
